Firmware

Complete revision history for ASU-1 firmware. Install only via authenticated USB-C connection. Do not interrupt firmware installation. Anomalous behavior following a firmware update should be reported to support immediately.

Release Date 2024-09-03
Compatible Hardware ASU-1 Rev. B, Rev. C

v4.1.7 2024-09-03 Stable
  • Improved Band A frequency step stability at temperature extremes below −20°C
  • Corrected LED 3 indicator flicker during high-throughput write operations
  • Adjusted night mode minimum brightness from 0.8 nit to 0.5 nit per field operator feedback
  • Battery calibration algorithm update — improves duration estimate accuracy by ~11%
  • Minor display rendering correction for directional array at high refresh rates
v4.1.4 2024-05-17 Stable
  • Resolved intermittent Band B dropout when operating simultaneously with Band A above 480 MHz
  • R5/R6 secondary rotary controls now configurable via USB-C parameter file
  • Push button B3 default function reassigned from display brightness to Band A/B phase alignment
  • Internal storage encryption key rotation interval extended from 24hr to 72hr — reduces wear on storage controller
  • Startup POST sequence shortened by approximately 1.2 seconds
v4.0.9 2024-01-08 Critical
  • Security patch: addressed authenticated session vulnerability in USB-C transfer protocol
  • EMIT control behavior correction — see anomaly note below
  • Corrected unintended EMIT activation under specific Band A + Band B simultaneous conditions. Prior behavior: unit would activate EMIT autonomously when both acquisition bands registered signal patterns matching internal pattern table. This was not intended behavior. The pattern table has been isolated. Autonomous EMIT activation is no longer possible.
  • All units produced before 2023-11-01 should update immediately
Anomaly Report · AR-0017 Eleven field operators reported that their units activated EMIT without manual input between October and December 2023. The triggering condition has been identified and corrected in v4.0.9. Operators who experienced this behavior are encouraged to submit a support ticket referencing AR-0017. We are still evaluating whether the pattern table that triggered this behavior was present in the factory firmware or introduced via a prior update. Investigation ongoing.
v4.0.2 2023-08-14 Stable
  • Major version: complete acquisition stack rewrite for Rev. C hardware
  • Dual-band simultaneous operation enabled (previously sequential only)
  • New display rendering engine — 40% reduction in power draw for display subsystem
  • LSOD control implemented (Loss-of-Signal Override Disable) — see Section 4 of manual
  • EMIT control implemented — see Section 4 of manual
  • Removed legacy Band A preset configuration mode (deprecated since v3.2)
  • Startup sequence revised: POST now tests EMIT subsystem integrity on each boot
v3.4.1 2022-11-30 Stable
  • Final v3 release for Rev. B hardware
  • Battery duration improvements — LiFePO₄ discharge curve refinement
  • Corrected edge case in frequency tuning logic that caused R1 to skip steps at certain gain settings
  • Audio output routing now retains state across power cycles
v3.1.0 2022-02-07 Stable
  • First public firmware release
  • Band A single-channel acquisition, frequency configurable via R1
  • Circular OLED display initialization and directional array rendering
  • Internal storage with AES-256 encryption
  • USB-C data transfer and firmware update pathway
  • Three additional subsystems initialized in this version are not described here. They are described in Section 4 of the operating manual.
